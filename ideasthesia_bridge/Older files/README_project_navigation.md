This README explains what the three support documents are for, in what order to read them, and how they relate to the canonical project materials.

# Support documents

These are not the canon itself. They are support tools for navigating the current state of the project.

## 1) `comparative_audit_memo.md`

### What it is for
Use this file to understand the **current alignment status** of the public-facing pages against the canon.

### It tells you
- what is now aligned,
- what remains optional,
- what remains open,
- and what no longer needs to be reopened casually.

### Read this when
- you are returning after a break,
- you are unsure what has already been fixed,
- or you need a quick state-of-the-project snapshot.

---

## 2) `decision_ledger.md`

### What it is for
Use this file to distinguish between:
- what has been **settled**,
- what is still **open**,
- and what is intentionally **provisional**.

### It tells you
- which questions are already decided,
- which decisions still need to be made,
- and which things should not be re-debated unless the project changes scope.

### Read this when
- something suddenly feels unresolved again,
- you are tempted to reopen an old question,
- or you need to know whether an issue is genuinely open or just annoying.

---

## 3) `editorial_alignment_handout.md`

### What it is for
Use this file as the practical **work plan**.

### It tells you
- what still needs editing,
- what order to do things in,
- and what the recommended next passes are.

### Read this when
- you are ready to edit,
- you need a sequence of practical steps,
- or you want the shortest path back into hands-on project work.

---

# Recommended reading order on re-entry

When returning to the project after time away, read the support files in this order:

## Step 1 — Read `comparative_audit_memo.md`
This tells you the current state.

## Step 2 — Read `decision_ledger.md`
This tells you what is settled, open, or provisional.

## Step 3 — Read `editorial_alignment_handout.md`
This tells you what to do next.

---

# Fast orientation rule

If you are tired, overwhelmed, or unsure where to begin:

- **Audit memo** = where things stand
- **Decision ledger** = what is actually open
- **Editorial handout** = what to do next

---

# Simple mnemonic

## State → Decision → Action

- **State** = `comparative_audit_memo.md`
- **Decision** = `decision_ledger.md`
- **Action** = `editorial_alignment_handout.md`

# Closing note to future self

Do not begin by reopening everything.

Begin by locating the canon, checking the state, consulting the decisions, and only then moving into action.

The project does not need to be rediscovered from scratch each time you return.
